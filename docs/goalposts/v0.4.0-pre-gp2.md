# Goalpost v0.4.0-Pre-GP2: Completion Integrity Gate

## Description

Before starting `bunny-codec`, audit every previously completed roadmap claim
against source code, tests, CI, and goalpost documents. Completed labels must
mean that the written acceptance criteria are actually satisfied by
implementation evidence, not softened after the fact.

## Status

Completed

## Acceptance Criteria

* Produce an evidence-backed completed-claim audit that lists verified claims,
  partial claims, and unsubstantiated claims.
* Update `ROADMAP.md`, `docs/BEARING.md`, and affected goalpost documents so
  completed claims identify any missing implementation evidence.
* Identify every outstanding acceptance criterion that must be finished before
  v0.4.0-GP2 can begin.
* Convert each outstanding criterion into one or more follow-up finish-off
  slices.
* Keep v0.4.0-GP2 blocked until all finish-off slices are complete.

### Partial Claims Found

These claims were previously marked complete but were only partially backed at
the start of this gate. Each item was converted into a finish-off slice and is
verified above.

| Claim Area | Partial Evidence Found | Final Disposition |
| --- | --- | --- |
| v0.1.1-GP1 Directive-Driven Scalar Mapping | Scalar mapping was partially directive-aware, but unprofiled custom scalars could still fall back to host placeholder types. | Enforced mandatory `@bunnyScalarProfile`, centralized profile lookup, and added rejection/source-guard tests. |
| v0.1.1-GP3 Headless WebAssembly Verification | CI covered most WASM-compatible library crates but omitted `bunny-contract`, while older wording implied full workspace WASM execution. | Added `bunny-contract` headless WASM coverage and documented host-side crate exclusions. |
| v0.2.0-GP1 Core Bounding Shapes | Float-to-fixed conversion wording implied infallible `From`/`Into`, but safe ingress is fallible. | Added explicit validating `try_new`, `try_into_fixed`, `try_from_float`, and `TryFrom` APIs with rejection tests. |
| v0.2.0-GP1 Normalized Wrappers | Runtime unit-vector normalization existed, but compile-time proof support was not complete. | Added const `try_from_unit` APIs, axis constants, and native/WASM compile-time proof tests. |
| v0.2.0-GP2 Ray-Casting Queries | Ray solvers had functional tests, but the fixed-seed corpus and raw-output determinism gate were incomplete. | Added seeded corpus generation plus raw Q32.32 expected-output assertions for sphere, AABB, and triangle rays. |
| v0.2.0-GP3 Closest Point Queries | Closest-point tests leaned on `to_f32()` projections rather than byte-level raw assertions. | Added raw Q32.32 assertions for AABB, AABB/sphere, triangle, and segment cases. |
| v0.3.0-GP1 Stable BVH Tree | BVH implementation existed, but malformed input and zero-allocation claims needed stronger proof. | Replaced panic-capable internals with checked access, added malformed-buffer tests, and added a native counting-allocator witness. |
