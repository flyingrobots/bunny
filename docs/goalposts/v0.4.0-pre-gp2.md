# Goalpost v0.4.0-Pre-GP2: Completion Integrity Gate

## Description

Before starting `bunny-codec`, audit every previously completed roadmap claim
against source code, tests, CI, and goalpost documents. Completed labels must
mean that the written acceptance criteria are either actually satisfied or
corrected to match verified reality.

## Status

Active

## Acceptance Criteria

* Produce an evidence-backed completed-claim audit that lists verified claims,
  partial claims, and unsubstantiated claims.
* Update `ROADMAP.md`, `docs/BEARING.md`, and affected goalpost documents so
  completed claims do not overstate the implementation.
* Identify every outstanding acceptance criterion that must be finished before
  v0.4.0-GP2 can begin.
* Convert each outstanding criterion into one or more follow-up finish-off
  slices.
* Keep v0.4.0-GP2 blocked until all finish-off slices are complete.
