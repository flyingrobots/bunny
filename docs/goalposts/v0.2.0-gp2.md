# Goalpost v0.2.0-GP2: Ray-Casting Queries

## Description

Implement deterministic ray-intersection math solvers in `bunny-query`.

## Status

Completed

## Acceptance Criteria

* Ray-Sphere intersection solver returns hit coordinate and normal or None.
* Ray-AABB intersection solver returns enter/exit distances or None.
* Ray-Triangle intersection solver (e.g., Möller–Trumbore algorithm) returns intersection coordinate or None.
* Determinism: Given a fixed test corpus and fixed RNG seeds, the ray-intersection solvers produce identical outputs (bitwise identical or within a defined epsilon tolerance) across all CI runners and supported platforms (Linux, macOS, Windows). This is validated via a canonical test suite executed in CI that fails on any divergence.

## Completion Evidence

* `crates/bunny-query/tests/ray_determinism_tests.rs` defines the canonical
  `RAY_DETERMINISM_CORPUS_SEED` and raw Q32.32 expected outputs for
  ray-sphere, ray-AABB, and ray-triangle hit and miss cases.
* The corpus asserts raw integer outputs, not `to_f32()` projections, so native
  and WASM runners fail on any byte-level divergence.
* The test is a normal `bunny-query` integration test with
  `wasm_bindgen_test(unsupported = test)`, so it runs through workspace native
  CI on Linux, macOS, and Windows, and through the existing
  `wasm-pack test --node crates/bunny-query --locked` CI gate.
