# Goalpost v0.4.0-GP2: File Format Adapters

## Description

Provide zero-copy mesh format parsers in `bunny-codec`.

## Status

Active

## Acceptance Criteria

* Zero-copy PLY binary parser.
* Zero-copy OBJ text parser.
* Regression fixtures using Stanford Bunny sample meshes.

## Completion Evidence

* Slice 2.1: `bunny-codec` now provides `parse_binary_ply`, which parses the
  canonical binary little-endian PLY mesh profile as a borrowed
  `PlyBinaryMesh` view over the original input bytes.
* The PLY parser validates canonical `float x/y/z` vertex properties and
  `property list uchar int vertex_indices` triangle faces, rejects unsupported
  formats/layouts, and exposes typed vertex/triangle accessors without copying
  the vertex or face payload sections.
* Native tests include a counting-allocator witness that `parse_binary_ply`
  performs zero heap allocations on the accepted path. Native and WASM tests
  cover valid parsing, borrowed payload slices, non-binary format rejection,
  non-triangle face rejection, negative index rejection, and trailing-payload
  rejection.
