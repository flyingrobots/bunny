# Goalpost v0.4.0-GP2: File Format Adapters

## Description

Provide zero-copy mesh format parsers in `bunny-codec`.

## Status

Completed

## Acceptance Criteria

* Zero-copy PLY binary parser.
* Zero-copy OBJ text parser.
* Regression fixtures using Stanford Bunny sample meshes.

## Completion Evidence

* Slice 2.1: `bunny-codec` now provides `parse_binary_ply`, which parses the
  canonical binary little-endian PLY mesh profile as a borrowed
  `PlyBinaryMesh` view over the original input bytes.
* The PLY parser validates canonical `float x/y/z` vertex properties and
  `property list uchar int vertex_indices` triangle faces, rejects unsupported
  formats/layouts, non-finite vertex coordinates, and out-of-bounds face
  indices, and exposes typed vertex/triangle accessors without copying the
  vertex or face payload sections.
* Native tests include a counting-allocator witness that `parse_binary_ply`
  performs zero heap allocations on the accepted path. Native and WASM tests
  cover valid parsing, borrowed payload slices, non-binary format rejection,
  non-triangle face rejection, negative index rejection, out-of-bounds face
  index rejection, non-finite vertex rejection, and trailing-payload rejection.
* Slice 2.2: `bunny-codec` now provides `parse_obj_text`, which parses a
  supported OBJ mesh profile as a borrowed `ObjMesh` view over the original
  source string.
* The OBJ parser validates `v x y z` vertex records, triangular `f` face
  records, finite vertex coordinates, one-based OBJ indices, slash-form face
  tokens, and face-index bounds. It exposes typed vertex/triangle accessors
  without copying source records.
* Native tests include a counting-allocator witness that `parse_obj_text`
  performs zero heap allocations on the accepted path. Native and WASM tests
  cover valid parsing, borrowed source retention, slash-form faces,
  non-triangle face rejection, out-of-bounds index rejection, relative/zero
  index rejection, non-finite vertex rejection, and unsupported statement
  rejection.
* Slice 2.3: `crates/bunny-codec/tests/fixtures/stanford_bunny_subset.obj`
  records a tiny Stanford Bunny OBJ subset with source URL, source vertices,
  and source face provenance. `stanford_fixture_tests.rs` parses it through
  `parse_obj_text` and asserts the expected vertex/triangle outputs.
* The same fixture test module includes canonical binary PLY bytes derived from
  the official Stanford 3D Scanning Repository `bun_zipper.ply` source vertices
  and first source face, then parses those bytes through `parse_binary_ply` and
  asserts the expected vertex/triangle outputs.
