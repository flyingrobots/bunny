# Goalpost v0.1.1-GP1: Directive-Driven Scalar Mapping

## Description
Refactor the code generator to dynamically resolve custom scalar profiles from schema directives instead of hardcoded name strings.

## Status
Completed

## Acceptance Criteria
* `bunny-wesley` retrieves and parses the `@bunnyScalarProfile` directive arguments.
* Scalar mappings (like `BunnyFixedQ32_32Raw` -> `i64`) are driven by directive arguments.
* All hardcoded scalar name strings in `bunny-wesley/src/main.rs` must be removed or replaced with well-named constants or configuration values (no literal scalar-name string occurrences remain). The code compiles, unit/integration tests pass, and verification is performed by ensuring zero literal scalar-name matches remain.
