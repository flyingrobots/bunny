# Goalpost v0.1.1-GP1: Directive-Driven Scalar Mapping

## Description

Refactor the code generator to dynamically resolve custom scalar profiles from schema directives instead of hardcoded name strings.

## Status

Completed

## Acceptance Criteria

* `bunny-wesley` retrieves and parses the `@bunnyScalarProfile` directive arguments.
* Scalar mappings (like `BunnyFixedQ32_32Raw` -> `i64`) are driven by directive arguments.
* Custom scalar definitions without `@bunnyScalarProfile` are rejected instead
  of silently falling back to host-language placeholder types.
* Supported scalar profile mappings are centralized in an explicit registry
  instead of ad hoc scalar-name branches.
* Source-level tests guard against reintroducing legacy hardcoded scalar-name
  branches in `bunny-wesley`.
