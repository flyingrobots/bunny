# Goalpost v0.1.1-GP3: Headless WebAssembly Verification

## Description

Upgrade the CI workflow to execute unit tests inside actual headless WebAssembly environments.

## Status

Completed

This is a historical goalpost document. References to `ci.yml` describe the
workflow used when the goalpost was completed; current verification runs through
`.github/workflows/code-dojo.yml`.

## Acceptance Criteria

* Configure GitHub Actions to execute the unit test suite for every
  WASM-compatible library crate inside a headless Node.js/V8 WASM runner via
  `wasm-pack test`.
* Keep host-side binary/tooling crates covered by native workspace tests when
  they are not WASM library crates.

## Completion Evidence

* `ci.yml` runs `wasm-pack test --node --locked` for every WASM-compatible
  library crate: `bunny-num`, `bunny-linalg`, `bunny-geom`,
  `bunny-contract`, `bunny-query`, `bunny-broadphase`, `bunny-mesh`, and
  `bunny-codec`.
* `bunny-wesley` and `xtask` are intentionally excluded from `wasm-pack test`
  because they are host-side binary/tooling crates, not WASM library crates.
  They remain covered by native workspace tests.
