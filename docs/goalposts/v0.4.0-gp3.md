# Goalpost v0.4.0-GP3: Compression Decoders

## Description

Define and implement a deterministic decoder for a canonical Bunny compressed
mesh byte stream inside `bunny-codec`.

## Status

Implemented and verified locally. PR #105 has green GitHub CI, CodeRabbit
approval, and no inline review threads; it is awaiting the merge gate.

## Problem

GP1 introduced stable quantized mesh buffers in `bunny-mesh`. GP2 introduced
external file format adapters in `bunny-codec`. GP3 closes the mesh commons
track by adding a Bunny-native compressed byte profile that reconstructs
validated quantized mesh records without depending on OBJ, PLY, glTF, STL, or
other external format semantics.

## Non-Goals

* Do not add new OBJ, PLY, glTF, STL, or other external format profiles.
* Do not add compression encoders in this goalpost.
* Do not add transform, animation, material, texture, or scene graph semantics.
* Do not weaken existing GP2 zero-copy parser claims.
* Do not claim typed slice borrowing over arbitrary input bytes. Rust cannot
  safely reinterpret arbitrary `&[u8]` as `&[QuantizedVertex]` or triangle
  slices without `unsafe`, and this repository forbids unsafe code.
* Do not include a checksum in profile version 1. Content verification remains
  the responsibility of `bunny-mesh` hashing over decoded records and bounds.

## Canonical Byte Profile

The GP3 decoder profile is Bunny-native and decodes into existing `bunny-mesh`
record types through checked accessors.

All integer fields are little-endian.

| Offset | Size | Field | Rule |
| ---: | ---: | --- | --- |
| 0 | 8 | Magic | ASCII `BUNNYQZ!` |
| 8 | 1 | Version | `1` |
| 9 | 1 | Index width | `16` or `32` |
| 10 | 2 | Flags | Must be `0` |
| 12 | 4 | Vertex count | `1..=1_000_000` |
| 16 | 4 | Triangle count | `1..=1_000_000` |
| 20 | 8 | Payload length | Must equal canonical payload size |
| 28 | 48 | Bounds | Six Q32.32 raw `i64` values: min xyz, max xyz |
| 76 | N | Vertex payload | `vertex_count * 6` bytes |
| 76 + N | M | Triangle payload | `triangle_count * stride(index_width)` bytes |

The vertex payload stores each quantized vertex as three little-endian `u16`
values: `x`, `y`, and `z`.

The triangle payload stores each face as either:

* Width 16: three little-endian `u16` vertex indices.
* Width 32: three little-endian `u32` vertex indices.

For width 16, `vertex_count` must be at most `65_536`, because no `u16` index
can address vertices beyond that range. For both widths, every decoded triangle
index must be `< vertex_count`.

Bounds use `FixedAabb3::try_new`; inverted min/max bounds are rejected. Flat
axes are permitted because existing quantization helpers already define
deterministic behavior for zero-span axes.

## Allocation Contract

`decode_compressed_mesh` returns a borrowed `CompressedMesh<'_>` view. The view
borrows the vertex and triangle byte sections from the caller-owned input,
validates all indices before returning, and exposes typed `bunny-mesh` records
through checked accessors.

The accepted path performs zero heap allocations after warm-up. This is proved by
the native allocation witness. The decoder deliberately does not expose typed
slices because doing that over arbitrary byte input would require `unsafe`.

## Canonical Errors

The public error model is:

* `InvalidMagic`
* `UnsupportedVersion`
* `UnsupportedFlags`
* `InvalidIndexWidth`
* `InvalidCount`
* `InvalidBounds`
* `InvalidPayloadLength`
* `PayloadTooShort`
* `TrailingData`
* `IndexOutOfBounds`
* `IntegerOverflow`

The malformed-input corpus covers every public error variant.

## Acceptance Criteria

* A documented Bunny compressed mesh byte profile exists in this goalpost.
* `bunny-codec` exposes `decode_compressed_mesh`, `CompressedMesh`,
  `CompressedIndexWidth`, `CompressedTriangle`, and `CompressedMeshError`.
* The decoder returns a validated borrowed byte view with typed checked accessors.
* The decoder rejects unsupported versions, invalid lengths, invalid counts,
  invalid quantization bounds, invalid indices, non-canonical payload encodings,
  integer overflow, and trailing bytes.
* Native tests cover width-16 and width-32 accepted paths, every public error
  variant, pointer borrowing, raw fixture stability, and out-of-range accessors.
* WASM tests cover the same public compressed decoder corpus via
  `wasm_bindgen_test`.
* Native allocation witnesses prove the accepted-path zero-allocation contract.
* `CHANGELOG.md`, `ROADMAP.md`, and this goalpost document record final evidence
  before GP3 is marked completed.
