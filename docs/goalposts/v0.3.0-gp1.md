# Goalpost v0.3.0-GP1: Stable BVH Tree

## Description

Build a memory-stable, zero-allocation bounding volume hierarchy (BVH) inside `bunny-broadphase`.

## Status

Completed

## Acceptance Criteria

* Stable array-backed tree layout requiring zero runtime heap allocation.
* SAH (Surface Area Heuristic) tree building solver.
* Deterministic tree traversal for box overlap and ray intersection.

## Completion Evidence

| Acceptance Criterion | Evidence |
| --- | --- |
| Stable array-backed tree layout requiring zero runtime heap allocation. | `BvhNode` stores the flat node layout, and `build_bvh` writes into caller-owned node and primitive-index slices. `crates/bunny-broadphase/tests/bvh_allocation_tests.rs` installs a native counting allocator and proves `build_bvh`, `intersect_aabb`, and `intersect_ray` perform zero heap allocations on that caller-owned API surface. |
| SAH (Surface Area Heuristic) tree building solver. | `build_bvh` evaluates centroid bounds, candidate split axes, split coordinates, and surface-area costs before partitioning primitives into child nodes. |
| Deterministic tree traversal for box overlap and ray intersection. | `intersect_aabb` and `intersect_ray` use fixed-size stack traversal over the flat node array and deterministic primitive-index order. The native allocation witness exercises both traversal paths without heap allocation. |
| Panic-free malformed input handling. | `build_bvh` validates buffer sizes with checked arithmetic and checked node/primitive access helpers; malformed builder buffers return `None`. BVH traversal reports malformed node indices or primitive ranges through `TraversalError`. `crates/bunny-broadphase/tests/bvh_builder_tests.rs` uses `wasm_bindgen_test(unsupported = test)`, so malformed builder-buffer assertions run in both native tests and the headless WASM gate. |
