# Bunny — Executive Summary

## Identity

Bunny is a neutral Rust graphics commons designed to establish absolute, bit-level CPU mathematical determinism for graphics primitives, linear algebra, geometries, and DTO contracts.

Bunny operates under a strict project-neutral boundary:
* Bunny core crates do not import, depend on, or learn about Echo transaction strands, Geordi render backend receipts, jedit UI workflows, or Unity/browser environment contexts.
* Downstream runtimes (Echo, Geordi, jedit/Jim) consume Bunny as their core graphics math substrate.

## Current State

The algebraic foundation is established. Bunny implements type-safe fixed-point representation (`FixedQ32_32`) with saturating math and Ties-to-Even rounding, alongside 2D/3D vectors (`FixedVec2`, `FixedVec3`) and a deterministic integer square root. Contracts are autogenerated from schemas via the `bunny-wesley` compiler.

## Legends

### MATH
Core fixed-point profile design, numeric conversions, rounding, and saturating math operations.
* **Active**: Implementation of `FixedQ32_32::sqrt()` for geometric operations.

### LINALG
Fixed-point linear algebra vector operations (`dot`, `cross`, `normalize`).
* **Active**: Implementations of `FixedVec2` and `FixedVec3`.

### GEOM
Geometric bounding envelopes and shapes (`FixedRay3`, `FixedAabb3`, `FixedSphere3`).
* **Active**: v0.2.0 Core Bounding Shapes.

### COMPILER
GraphQL SDL code generator extension (`bunny-wesley`).
* **Active**: Decoupling hardcoded scalar profiles using AST directives.

### INFRA
CI pipelines, Clippy enforcement, and target compilation gates.
* **Active**: GitHub Actions cross-platform validation.
